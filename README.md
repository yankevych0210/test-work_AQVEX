# AQVEX — Каталог Продукции (Тестовое Задание)

Добро пожаловать в репозиторий клиентской части каталога продукции **AQVEX**. 
Приложение реализовано в виде Single Page Application (SPA), где основной упор сделан на качество кода, компонентную архитектуру и полное соответствие макетам.

## 🚀 Предпросмотр и Запуск

### Требования
- Node.js (рекомендуется v18+ или v20+)
- npm или yarn

### Установка и локальный запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yankevych0210/test-work_AQVEX.git
   cd aqvex-catalog
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
Приложение будет доступно по адресу `http://localhost:5173`. 
*(API-запросы настроены на проксирование прямо через `vite.config.ts`, чтобы обойти проблемы CORS).*

### Сборка для Production
```bash
npm run build
```
Готовые статические файлы будут лежать в папке `dist/`.

## 🛠 Стек Технологий

Проект использует современный и популярный стек технологий для обеспечения стабильности и быстрой работы:

- **React 19** — Создание реактивных UI-компонентов.
- **TypeScript** — Строгая статическая типизация для предотвращения багов на этапе разработки.
- **Vite** — Невероятно быстрая сборка и HMR (Hot Module Replacement) сервер.
- **SCSS Modules (`.module.scss`)** — Локальная область видимости стилей (изоляция) с поддержкой вложенности и переменных. Не нужно тянуть тяжелые библиотеки.
- **React Hooks (Custom Hooks)** — Изоляция бизнес-логики от UI слоя.

## 🏗 Архитектура Проекта

Структура `src/` разделена по функциональным зонам для масштабируемости:

```text
src/
├── assets/          # Иконки, логотипы, локальные изображения.
├── components/      # UI-компоненты (Reusable). Каждый компонент имеет изолированную папку:
│   ├── Header/        # Шапка (Лого + Поиск)
│   ├── SortBar/       # Сортировка и количество товаров
│   ├── ProductGrid/   # CSS-Grid сетка списка товаров
│   ├── ProductCard/   # Отдельная карточка товара
│   ├── Rating/        # Компонент рейтинга (SVG звезды)
│   ├── VolumeSelect/  # Селект обьема товара (ml / g)
│   ├── Pagination/    # Постраничная навигация
│   ├── Skeleton/      # Анимация загрузки (Shimmer effect)
│   └── Footer/        # Подвал (Информация и логотипы платежек)
├── hooks/           # Кастомные React Хуки
│   └── useProducts.ts # Обертка на логику загрузки, сортировки, пагинации и поиска товаров
├── services/        # Слой для работы с внешним миром (API)
│   └── api.ts         # Нативные Fetch-запросы и их типизация
├── types/           # Глобальные TS интерфейсы (Domain Models)
│   └── product.ts     # Интерфейсы: Product, Volume, SortOption
├── utils/           # Утилиты
│   └── debounce.ts    # Задержка ввода для поиска 
├── App.tsx          # Точка сборки приложения
└── index.scss       # Глобальные стили (Vars, CSS Reset)
```

**Основные архитектурные решения:**
1. **Separation of Concerns**: UI отделён от бизнес-логики. Компоненты (`ProductCard`, `Pagination`) ничего не знают о том, откуда берутся данные. Вся работа с состояниями вынесена в `useProducts.ts`.
2. **Barrel Exports**: Для удобных импортов в папках с компонентами используются файлы `index.ts`.
3. **Graceful Degradation for Images**: Если у товара битая ссылка на фотографию — изящно показывается falllback-плейсхолдер (заглушка).

## ✨ Реализованный Функционал

### 1. Загрузка и Отображение товаров
- Получение данных по REST API (`GET /api/v1/products`).
- Состояния загрузки (`isLoading`): Реализован красивый Skeleton UI Loader для карточек товаров.
- Состояние ошибки (`error`): Информативное уведомление, если сервер не отвечает.

### 2. Сортировка (`Select`)
- По популярности (по убыванию количества отзывов и рейтинга).
- От дешевых к дорогим (Сначала низкая цена).
- От дорогих к дешевым (Сначала высокая цена).
- По названию (Сортировка по алфавиту).

### 3. Живой Поиск (Search)
- Выполняется поиск по названиям продуктов и категориям.
- Использован **Debounce** (300ms) для предотвращения спам-перерисовок (или спама API) при быстром вводе текста пользователем.

### 4. Пагинация (Pagination)
- Корректно разбивает массив данных (по умолчанию 12 элементов на страницу).
- При вводе текста в поиск, пагинация автоматически сбрасывается на первую страницу во избежание бага с "виртуальными" пустыми страницами.
- Задизайнена под макет: выделяется текущая страница, скрываются лишние кнопки при большом количестве страниц (эллипсис `...`).

### 5. Адаптивность (Responsive Design)
Макет автоматически трансформируется от десктопа к смартфонам с использованием CSS Grid:
- `> 1024px` — 4 колонки
- `769px–1024px` — 3 колонки
- `481px–768px` — 2 колонки
- `≤ 480px` — 1 колонка

## 🎨 Разработка UI (Pixel Perfect)

Компоненты (карточка товара, кнопка корзины, селект сортировки, скидочные бейджи) были детально стилизованы в соответствии с Figma:
- Формы кнопок, обводки, отступы.
- "Лейбы" скидок (отрезаны углы с помощью `clip-path`, перфорация).
- Линейные градиенты там, где они требуются в макете.
- Интерактивность: легкий эффект при наведении (hover) и нажатии (active) на кнопки и карточки.
